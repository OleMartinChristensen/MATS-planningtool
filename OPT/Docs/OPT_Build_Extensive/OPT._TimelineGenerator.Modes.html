OPT._TimelineGenerator.Modes package¶

Contains modules which correspond to the scheduling of each Mode.

Schedules and calculates duration of modes.

The top or core function of each module is the only one called from higher levels. The name of the core function in each Mode specific module must be equal to the name of the Mode as stated in Modes_priority found in the Configuration File.

The core function may call two other functions inside the same module, date_calculator() and date_select. date_calculator returns calculated or chosen dates (1 or many). date_select chooses the most appropriate date or/and makes sure the date chosen is available.

Core function for the scheduling of a Mode.

Arguments:

Occupied_Timeline (dict of list): Dictionary with keys equal to planned and scheduled Modes with entries equal to their start and end time as a list.

Returns:

(dict of list): Occupied_Timeline (updated for the current Mode).

(str): Comment regarding the scheduling result of the active mode.

OPT._TimelineGenerator.Modes.Mode124.date_calculator()[source]¶

Subfunction, Simulates MATS FOV and the Moon.

Determines when the Moon is entering the FOV at an vertical offset-angle equal to #V-offset and also being located at a horizontal off-set angle equal to less than #H-offset when pointing at an LP altitude equal to #pointing_altitude.

A timeskip equal to 1/2 an orbit of MATS is applied after a successful spotting of the Moon to save processing time.

A timeskip equal to the time it takes for the Moon orbital position to change by #H-offset/4 degrees are also applied if the Moon is determined to be at an horizontal off-set angle larger then the horizontal FOV of the instrument, equal to #HFOV.

(# as defined in the Configuration File).

Saves the date and parameters regarding the spotting of the Moon Also saves relevant data to an .csv file located in Output/.

Arguments:

Returns

A list containing dictionaries containing parameters for each time the Moon is spotted. Or just a date depending on 'automatic' in Mode124_settings.

Return type

SpottedMoonList ((list of dict)) or (str)

OPT._TimelineGenerator.Modes.Mode1_2_5 module¶

Schedules Operational Science Modes wherever time is available as defined by the Occupied_Timeline dictionary.

Part of Timeline_gen, as part of OPT. Operational Science Modes is always scheduled after the rest of the planned Modes have been scheduled. Result is saved in the Occupied_Timeline dictionary.

OPT._TimelineGenerator.Modes.Mode1_2_5.Mode1_2_5(Occupied_Timeline)[source]¶

Core function for the scheduling of Operational Science Modes (Mode 1, 2, and 5).

Which Mode that will be scheduled depends on Timeline_settings['Choose_Operational_Science_Mode']. These Modes, called Operational Science Modes, are always scheduled last, and wherever time is available. Only one Operational Science Mode is scheduled for each timeline.
